```markdown
## 1. الهدف التجاري
بناء منصة وطنية موحدة لإدارة المسوحات الرقمية وتحويل البيانات التشغيلية إلى رؤى تنفيذية دقيقة تدعم اتخاذ القرار.

- تشتت بيانات المسوحات بين أنظمة متعددة.

## 3. الأهداف
- تقليل زمن إعداد التقرير بنسبة 60%.

- المستخدمون التنفيذيون وصانعو القرار.

## 5. نطاق الأعمال
### داخل النطاق
- إدارة المسوحات والملاحظات.
- لوحة مؤشرات موحدة.
- تصدير تقارير PDF/Excel.
- صلاحيات متعددة الأدوار.

### خارج النطاق
- أنظمة ERP المالية.
- إدارة الموارد البشرية.

## 6. مؤشرات النجاح
- تبني المستخدمين >80% خلال 3 أشهر.
```

```markdown
# PRD - Product Requirements Document

## 1. رؤية المنتج
منصة رقابية ذكية تُمكّن الجهات من مراقبة الأداء التشغيلي وتحسين الامتثال عبر رؤى لحظية.

## 2. شخصيات المستخدمين
- مدير تنفيذي: يريد مؤشرات دقيقة مختصرة.
- محلل بيانات: يحتاج بيانات تفصيلية وتصدير مرن.
- مشغل ميداني: يحتاج تحديث الحالات بسرعة.
- مسؤول نظام: يدير المستخدمين والصلاحيات.

## 3. الرحلات الأساسية
1. تسجيل الدخول → لوحة المؤشرات.
2. إنشاء مسح → إرفاق الأدلة → تحديث الحالة.
3. تحليل النتائج → توليد تقرير → مشاركة النتائج.

## 4. متطلبات الإصدار
- MVP: تسجيل الدخول، إدارة مسوحات، Dashboard، تصدير.
- Phase 2: توصيات ذكية، تنبيهات متقدمة، تكاملات إضافية.

## 5. معايير القبول
- كل رحلة أساسية تعمل End-to-End.
- لا توجد أخطاء حرجة (P1/P0) قبل الإطلاق.

```

```markdown
3. عرض لوحات تحليلية ورسوم بيانية تفاعلية.
4. إنشاء وتصدير التقارير.

- واجهة ويب React.
- واجهات REST/tRPC للعميل الداخلي والخارجي.
- قاعدة بيانات MySQL.

## 5. القيود
- الاعتماد على اتصال شبكة مستقر.
- التوافق مع متطلبات الأمن المؤسسي.
```
