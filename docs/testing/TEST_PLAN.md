# Test Plan

## 1. نطاق الاختبار
- واجهات المستخدم الأساسية.
- API endpoints الرئيسية.
- صلاحيات الوصول.
- التصدير والتقارير.

## 2. أنواع الاختبارات
- Unit Tests.
- Integration Tests.
- End-to-End (حرجة).
- Security & Permission Testing.
- Performance Smoke Tests.

## 3. بيئات الاختبار
- Local Dev.
- Staging (مطابق نسبيًا للإنتاج).

## 4. معايير القبول
- جميع اختبارات P0/P1 ناجحة.
- لا أعطال حرجة مفتوحة.
- فشل اختبارات غير حرجة ≤ 5% مع مبرر.
