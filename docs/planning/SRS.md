# SRS - Software Requirements Specification

## 1. المقدمة
هذه الوثيقة تحدد المتطلبات البرمجية الوظيفية وغير الوظيفية لمنصة Rasid.

## 2. المتطلبات الوظيفية عالية المستوى
1. تسجيل دخول وإدارة جلسات المستخدمين.
2. إدارة المسوحات والحالات والأدلة.
3. عرض لوحات تحليلية ورسوم بيانية تفاعلية.
4. إنشاء وتصدير التقارير.
5. إدارة المستخدمين والأدوار والصلاحيات.
6. تكامل API مع أنظمة خارجية.

## 3. المتطلبات غير الوظيفية
- الأداء: استجابة API < 2 ثانية لمعظم العمليات.
- الأمان: JWT، تشفير كلمات المرور، وتدقيق العمليات.
- التوسع: دعم نمو البيانات والمستخدمين أفقيًا.
- الإتاحة: 99.5% حد أدنى.
- القابلية للصيانة: توثيق شامل وتغطية اختبارية مقبولة.

## 4. واجهات النظام
- واجهة ويب React.
- واجهات REST/tRPC للعميل الداخلي والخارجي.
- قاعدة بيانات MySQL.

## 5. القيود
- الاعتماد على اتصال شبكة مستقر.
- التوافق مع متطلبات الأمن المؤسسي.
